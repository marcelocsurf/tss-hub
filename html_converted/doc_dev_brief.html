SECTION 2: ROLES & PERMISSIONS (Deployed — DO NOT Change)

Admin

Full system access. Creates coaches, manages all system settings.

Coordinator

Creates students, sends intake links, assigns camps/coaches.

CONSTRAINT: Cannot change student belt level.

Coach

Sees assigned students only. Access includes: surf profile, goals, safety alerts, progression status, session history, own notes.

CONSTRAINT: Cannot see contact info, logistics details, or other coaches' notes.

Assistant

Safety-critical info only: name, belt level, ocean level, allergies, injuries, emergency contact.

CONSTRAINT: No edit access. View only.

SECTION 3: THE CASCADE — Session Creation Flow (22 Steps)

This is THE core feature to rebuild. Each step filters the next. The flow is organized into three moments: CONTEXT (Steps 1-5), PLANNING (Steps 6-13), and CLOSE (Steps 14-22).

MOMENT 1 — CONTEXT (Steps 1-5)

Step 1: Student

Type: Dropdown selector

Display: Student name + current belt level

Auto-loads: belt_level, sequence, step, ocean_level, medical alerts, last_session date

Filtering: Shows only students assigned to this coach

Step 2: Training Venue

Type: Dropdown selector

Options: Beachbreak, Pointbreak, Reefbreak, White water, Training pool, Surf skate, Gym, Tatami, Video Analysis Room

Logic: If non-water venue selected → skip Step 3 (Ocean Conditions)

Step 3: Ocean Conditions

Type: Dropdown selector

Options: Flat, 1-2 feet, 3-4 feet, 4-6 feet, 6+ feet

Display: Only shown if water venue selected in Step 2

Safety Check: Crosses student belt_level + ocean_level with ocean_rules table → produces risk state

Safety Override Rule: If ocean_rules return "blocked" → Session does NOT cancel. Instead: (a) System switches venue options to dry-land only, (b) Filters drills to land-based only, (c) Pilar part is maintained

Step 4: Session Type

Type: Dropdown selector

Options: Training (default), Competition, Evaluation

Step 5: Date

Type: Date picker

Default: Today's date

MOMENT 2 — PLANNING (Steps 6-13)

Step 6: Pilar Part

Type: Dropdown selector (filtered)

Filtering: Filtered by student belt_level only

Content: Shows Technical + Tactical parts appropriate for that belt level

Important: Coach does NOT select a pilar first. Coach goes directly to pilar part. System internally tags which pilar each part belongs to.

Special: Mental is implicit (Step 11). Physical is diagnostic (evaluation). Safety is automatic (Step 3).

Step 7: Mission

Type: Text field with AI suggestions

Coach writes: Session objective/goal

System can suggest: Based on pilar part + student history

Step 8: Drill

Type: Dropdown selector (filtered) with search

Filtering: Filtered by pilar_part + belt_level + venue + ocean_conditions

Display: Shows only relevant drills. Includes search box.

Step 9: Warm-up

Type: Dropdown selector

Options: Head to toe dynamic warm-up, Unnatural animal (for kids), Specific warm-up, Flow motion simulation

Step 10: Simulation

Type: Dropdown selector

Options: Surf skate, Flow motion, Visualization, Finger surfing the waves in front of you

Step 11: Mental Hack

Type: Dropdown selector

Options: Key words, Do it with intention, Breathe and reset, Positive self-talk, Visualize success, Process over outcome, One wave at a time

Step 12: Mission Time

Type: Dropdown selector

Options: 10 min, 15 min, 20 min, 30 min, 45 min, 60 min

Step 13: Repetitions

Type: Number input

MOMENT 3 — CLOSE (Steps 14-22, Mandatory)

Step 14: Status

Type: Dropdown selector

Options: Not Achieved, Partial, Competent, Mastered

Step 15: Focus Rating

Type: 1-5 button group

Applies to: All belt levels

Step 16: Frustration/Composure Rating

Type: 1-5 button group (progressive by belt level)

Rule: White/Yellow belts display Frustration (1-5) with label "Frustration"

Rule: Blue+ belts display Composure (1-5) with label "Composure"

Implementation: Check student.belt_level and display correct scale with correct label

Step 17: Coach Feedback

Type: Dropdown selector + text field

Quick Options: Not collaborating, Good effort, Very good, Excellent

Plus: Free text field for detailed feedback

Step 18: Achieved

Type: Text field

Coach documents: What student accomplished during session

Step 19: What's Next

Type: Dropdown selector (filtered)

Filtering: Same pilar part universe as Step 6

Purpose: Feeds continuity to next session

Step 20: Homework

Type: Dropdown selector + text field

Cues (predefined): Look forward, Weight on front foot, Knee in, Bend knees, Foot centered, Shoulders aligned with board nose, Activate scapula, Activate hands, Stay calm breath

Plus: Free text for custom homework

Step 21: Total Duration

Type: Dropdown selector

Options: 30 min, 45 min, 60 min, 90 min, 120 min

Step 22: Incident Report

Type: Optional toggle + conditional fields

If activated: Type (dropdown), Description (text), Action taken (text)

Type Options: Minor injury, Major injury, Equipment failure, Near miss, Other

SECTION 4: CRITICAL DATA DISTINCTION

students.belt_level

Definition: Current TRAINING level (what they are learning)

Critical: This is independent of what was purchased

student_level_access

Definition: What was PURCHASED (what they have access to)

Critical: Never conflate with belt_level. Example: A student can have purchased Blue Belt access but still be training at Yellow Belt.

students.ocean_level

Definition: Independent from belt_level

Determined by: Quiz (initial) + coach assessment (override)

Governs: Safety rules in ocean_rules matrix

student_path

Values: "Competitive" or "Recreational"

Affects: Which evaluation criteria display in Moment 3

Does NOT affect: Which sequences or drills are available

age_group

Values: "Adult" (14+) or "Junior" (8-14)

Affects: Drill variant suggestions and session_duration defaults

Does NOT affect: The progression sequence

SECTION 10: OCEAN RULES MATRIX

The ocean_rules table crosses belt_level with ocean_conditions to produce a risk state: "allowed", "caution", or "blocked".

* Yellow Belt at 3–4 feet: "caution" means allowed only if ocean_level (coach assessment) confirms the student can handle it. If no coach assessment on record, default to "blocked".

Safety Override: When "blocked" is returned → Session is NOT cancelled → System automatically switches venue options to dry-land only → Drills are filtered to land-based only.

SECTION 11: BOARD TRANSITION LOGIC

board_type is coach-assigned, NOT automatic by belt_level.

Softboard to Hardtop Transition Criteria

Student must demonstrate ALL of the following before transitioning to hardtop:

1. Effective turtle roll or duck dive

2. Left/right crossing while prone (paddling)

3. Left/right crossing while standing (riding)

4. Safe dismount (star fall) consistently

5. Coach sign-off recorded

Implementation

Add boolean field board_clearance_hardtop on student record.

Coach checks off criteria in a board transition checklist.

board_type dropdown shows all options, but system shows warning if board_clearance_hardtop is false and coach selects "Hardtop".
